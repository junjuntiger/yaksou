# 要件定義書 — 薬草みっけ

最終更新: 2026-06-25

---

## 1. プロジェクト概要

### 目的
身近な薬草・野草・花を記録・共有するアーカイブアプリを作る。
スマホで撮影するだけでAIが植物を識別し、効能や使い方を自動入力する。

### 背景
- 野草・薬草への関心があるが、知識を記録・整理する手段が少ない
- 撮った写真をただ保存するだけでなく、効能・使い方などの情報と紐づけたい
- 複数端末・複数ユーザーでデータを共有したい

---

## 2. 対象ユーザー

| ユーザー | 説明 |
|---------|------|
| 一般ユーザー | 薬草・野草に興味がある人。撮影・投稿・閲覧が目的 |
| 管理者 | アプリ全体の統計・登録者・投稿を管理する人 |

---

## 3. 機能要件

### 3-1. 認証

| # | 要件 |
|---|------|
| F01 | メールアドレス＋パスワードで新規登録できる |
| F02 | 登録済みアカウントでログインできる |
| F03 | ログアウトできる |
| F04 | ログイン中のみアプリを使用できる（未ログインはログイン画面のみ） |

### 3-2. 撮影・AI識別

| # | 要件 |
|---|------|
| F05 | スマホカメラまたはアルバムから写真を選択できる |
| F06 | 選択した写真をGemini AIに送信し、植物名・カテゴリ・効能・使い方を取得できる |
| F07 | AI識別結果を確認・編集してから保存できる |
| F08 | 植物が識別できない場合はエラーメッセージを表示する |

### 3-3. 植物記録

| # | 要件 |
|---|------|
| F09 | 植物名・カテゴリ・効能・使い方・地域・撮影日を記録できる |
| F10 | 保存した記録を一覧表示できる |
| F11 | 記録の詳細を表示できる |
| F12 | 自分が投稿した記録を修正できる |

### 3-4. みんなの投稿

| # | 要件 |
|---|------|
| F13 | 全ユーザーの投稿を一覧表示できる |
| F14 | カテゴリ（全て・薬草・花・雑草）でフィルタできる |
| F15 | 植物名・地域・撮影者名でキーワード検索できる |

### 3-5. マイ手帳

| # | 要件 |
|---|------|
| F16 | 自分の投稿のみを一覧表示できる |
| F17 | カテゴリ・キーワードでフィルタ・検索できる |

### 3-6. 管理

| # | 要件 |
|---|------|
| F18 | 総記録数・カテゴリ別件数・登録地域・最近の投稿を表示できる（統計） |
| F19 | 登録ユーザー一覧を表示できる |
| F20 | 全投稿を一覧表示できる（投稿管理） |

---

## 4. 非機能要件

| 項目 | 要件 |
|------|------|
| 対応デバイス | スマートフォン・PC（レスポンシブ対応） |
| 対応ブラウザ | Chrome最新版・Safari最新版 |
| 認証 | Firebase Authenticationを使用 |
| データ永続化 | Cloud Firestoreに保存（複数端末で同期） |
| AIレスポンス | 識別リクエストから10秒以内に結果を返す |
| セキュリティ | APIキーはサーバーサイドのみで管理 |

---

## 5. 対象外（スコープ外）

- 写真のクラウド保存（Firebase Storageは未設定）
- プッシュ通知
- ユーザー間のコメント・いいね機能
- オフライン対応
