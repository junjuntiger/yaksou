# データ設計書 — 薬草みっけ

最終更新: 2026-06-25

---

## 1. 使用データベース

**Cloud Firestore**（Firebase / Google Cloud）
- NoSQL ドキュメントデータベース
- リアルタイム同期（`onSnapshot` で変更を即時反映）
- 認証済みユーザーのみ読み書き可能

---

## 2. コレクション構造

```
Firestore
├── plants/                  ← 植物記録
│   └── {documentId}/        ← 自動生成ID（string）
│       ├── name             (string)
│       ├── category         (string)
│       ├── efficacy         (string)
│       ├── usage            (string)
│       ├── location         (string)
│       ├── date             (string)
│       ├── photographer     (string)
│       ├── userId           (string)
│       └── createdAt        (timestamp)
│
└── users/                   ← ユーザー情報
    └── {userId}/            ← Firebase Auth の UID と同じ
        ├── email            (string)
        ├── displayName      (string)
        ├── location         (string)
        └── createdAt        (timestamp)
```

---

## 3. plants コレクション

### フィールド定義

| フィールド名 | 型 | 必須 | 説明 | 例 |
|------------|---|------|------|----|
| `name` | string | ○ | 植物名（日本語） | `"ヨモギ"` |
| `category` | string | ○ | カテゴリ。薬草・花・雑草のいずれか | `"薬草"` |
| `efficacy` | string | - | 効能 | `"消炎・鎮痛・血行促進"` |
| `usage` | string | - | 使い方 | `"お灸・天ぷら・よもぎ茶"` |
| `location` | string | - | 撮影地域 | `"勝央町"` |
| `date` | string | ○ | 撮影日（YYYY-MM-DD形式） | `"2026-06-25"` |
| `photographer` | string | - | 撮影者の表示名 | `"薬草花子"` |
| `userId` | string | ○ | 投稿者のFirebase UID | `"abc123xyz"` |
| `createdAt` | timestamp | ○ | 作成日時（serverTimestamp） | — |

### インデックス
- 現在は特別なインデックスなし（Firestoreのデフォルト）
- 将来的に `userId` + `createdAt` の複合インデックスが必要になる可能性あり

## 4. users コレクション

### フィールド定義

| フィールド名 | 型 | 必須 | 説明 | 例 |
|------------|---|------|------|----|
| `email` | string | ○ | メールアドレス | `"user@example.com"` |
| `displayName` | string | - | 表示名（プロフィールで設定） | `"薬草花子"` |
| `location` | string | - | よく行く地域（投稿のデフォルト地域） | `"勝央町"` |
| `createdAt` | timestamp | ○ | 登録日時（serverTimestamp） | — |

### ドキュメントID
Firebase AuthenticationのUIDをそのまま使用。
`users/{uid}` の形式でアクセス。

## 5. データの読み書きフロー

### 植物記録の登録

```
ユーザーが保存ボタンを押す
    ↓
addDoc(collection(db, 'plants'), {
  name, category, efficacy, usage,
  location, date, photographer, userId,
  createdAt: serverTimestamp()
})
    ↓
Firestoreにドキュメント作成
    ↓
onSnapshotリスナーが変更を検知
    ↓
Reactのstateが更新 → 画面が自動更新
```

### リアルタイム同期

```
onSnapshot(collection(db, 'plants'), (snapshot) => {
  const records = snapshot.docs.map(doc => ({
    id: doc.id,
    ...doc.data()
  }))
  setPlants(records)
})
```

アプリ起動時にリスナーを登録し、データ変更があると即座に反映される。

---

## 6. 注意事項

- Firestoreは `undefined` 値を拒否するため、保存時は全フィールドに明示的な値またはフォールバックを設定する
- `id` フィールドはFirestoreが自動生成するstring型（旧仕様のnumber型から変更済み）
- 写真（画像データ）はFirestoreに保存できないため、現在は端末ローカルのBase64データのみ使用
- Firebase Storageは未設定（Blazeプランが必要）のため、画像の複数端末同期は未対応
