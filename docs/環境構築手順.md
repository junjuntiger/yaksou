## ローカル開発環境のセットアップ

### 1. リポジトリをクローン

```bash
git clone https://github.com/<your-username>/yaksou.git
cd yaksou
```

### 2. パッケージをインストール

```bash
npm install
```

### 3. 環境変数ファイルを作成

プロジェクトルートに `.env` ファイルを作成し、以下を記入：

```env
VITE_FIREBASE_API_KEY=your_firebase_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_project.firebasestorage.app
VITE_FIREBASE_MESSAGING_SENDER_ID=your_messaging_sender_id
VITE_FIREBASE_APP_ID=your_app_id
```

> Firebase設定値は [Firebaseコンソール](https://console.firebase.google.com/) → プロジェクト設定 → マイアプリ から取得。

> **注意**: `GEMINI_API_KEY` はVercelの環境変数に設定するため、`.env` には書かない。

### 4. 開発サーバーを起動

```bash
npm run dev
```

ブラウザで `http://localhost:5173` を開く。

## Firebase の設定

### Authentication の有効化

1. [Firebaseコンソール](https://console.firebase.google.com/) を開く
2. プロジェクトを選択 → 「Authentication」→「始める」
3. 「メール/パスワード」を有効化して保存

### Firestore の作成

1. 「Firestore Database」→「データベースの作成」
2. 本番環境モードで作成
3. 「ルール」タブで以下を設定して公開：

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
```

## Vercel へのデプロイ

### 初回設定

1. [Vercel](https://vercel.com/) にGitHubアカウントでログイン
2. 「Add New Project」→ GitHubリポジトリ（yaksou）を選択
3. 「Environment Variables」に以下を追加：

| 変数名 | 値 |
|--------|---|
| `GEMINI_API_KEY` | Google AI StudioのAPIキー |

4. 「Deploy」ボタンを押す

### 以降のデプロイ

```bash
git add .
git commit -m "変更内容"
git push origin main
```

GitHubにpushすると自動的にVercelがビルド・デプロイする。

## ローカルでVercel Functionsをテストする場合

```bash
npm install -g vercel
vercel dev
```

`http://localhost:3000` で起動。`/api/identify` が使えるようになる。
※ ローカルの `.env` に `GEMINI_API_KEY=your_key` を追加する必要あり。

## よくある問題

### `npm run dev` でエラーが出る

```bash
# node_modulesを削除して再インストール
rm -rf node_modules
npm install
```

### `.env` を更新したのに反映されない

Viteの開発サーバーを再起動する（Ctrl+C → `npm run dev`）。
環境変数の変更は再起動が必要。

### Firestoreに書き込めない

Firestoreのセキュリティルールを確認。
未ログイン状態では書き込めない仕様のため、ログインしてから操作すること。

### Vercelのデプロイが失敗する

VercelのDeploymentsページでビルドログを確認。
TypeScriptのエラーが原因のことが多い（`npm run build` をローカルで実行して確認）。
